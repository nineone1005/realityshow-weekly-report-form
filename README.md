# 综艺项目周报数据收集系统

这是一个用于综艺项目周报数据收集的表单系统，支持多方协作填写、上传截图/视频，并最终生成学城文档格式的周报。

## 文件说明

- `index.html` — 数据收集表单（供合作方填写）
- `review.html` — 数据审核页面（供审核人检查并生成文档）

## 使用流程

### 1. 合作方填写数据

打开 `index.html`，按模块填写数据：

1. **基本信息** — 项目名称、期数、播出平台、日期、项目背景、核心结论
2. **播出进度** — 播出状态、核心权益、12期播出计划表
3. **节目播出数据** — 播放量、热度、榜单数据、社媒指数、截图
4. **节目传播** — 热搜数据、短视频数据、宣推资源、爆款作品（支持截图/视频上传）
5. **节目权益表现** — 权益明细、客户权益秒数汇总、上刊视频/截图
6. **节目硬广** — 硬广类型、曝光数据、分端数据、截图
7. **品牌传播** — 微博/小红书/视频号传播、内部宣推、截图
8. **站内承接** — 搜索数据、站内承接数据表
9. **填写人信息** — 姓名、部门、联系方式

### 2. 保存草稿与提交

- 点击 **💾 保存草稿** — 数据自动保存到浏览器本地存储，刷新页面不会丢失
- 点击 **📤 提交数据** — 生成 JSON 文件并下载，发送给审核人

### 3. 进度条与邮件提醒

页面顶部有实时进度条，显示各模块填写完整度：

- 进度 **< 90%**：提示继续完善
- 进度 **>= 90%**：提交时自动触发提醒（弹窗提示审核人）

> 注意：当前版本为纯前端实现，邮件提醒通过弹窗模拟。如需真实邮件通知，需接入后端服务或企业微信/大象机器人。

### 4. 审核人生成学城文档

审核人打开 `review.html`，上传合作方提交的 JSON 文件：

1. 预览所有数据，检查无误
2. 点击 **📄 生成学城文档**
3. 系统自动生成 Markdown 格式文件并下载
4. 内容同时复制到剪贴板，可直接粘贴到学城文档中

## 数据字段映射

表单字段与学城文档结构的对应关系：

| 表单模块 | 学城文档章节 | 字段说明 |
|---------|------------|---------|
| 基本信息 | 文档标题、项目背景、核心结论 | projectName, episode, projectBackground, coreConclusion |
| 播出进度 | 播出进度表格 | airStatus, scheduleTable |
| 节目播出数据 | 节目播出 | totalPlayCount, heatPeak, thirdPartyTable, socialIndex |
| 节目传播 | 节目传播 | hotSearchTable, shortVideoTable, mediaResourceTable, hitWorksDesc |
| 节目权益表现 | 节目权益表现 | rightsOverview, rightsTable, clientSecondsTable |
| 节目硬广 | 节目硬广 | adTypes, adTotalExposure, adsTable |
| 品牌传播 | 品牌传播 | weiboSpread, xiaohongshuSpread, videoChannelSpread, tencentComments, internalPromotion |
| 站内承接 | 站内承接 | searchKeyword, searchQVTotal, siteTable |

## 扩展建议

1. **部署为内部服务**：可将 HTML 文件部署到公司内部静态页面服务，或封装为 NoCode 小程序
2. **接入真实邮件/大象通知**：在提交时调用企业邮件 API 或大象机器人接口发送通知
3. **图片/视频上传至对象存储**：当前文件仅保存在本地，可接入公司对象存储服务（如 WOS）获取永久 URL
4. **自动生成学城文档**：可进一步开发后端服务，直接调用学城 API 创建文档并插入图片

## 技术说明

- 纯前端实现，无需后端服务
- 使用 LocalStorage 保存草稿
- 支持响应式布局，适配移动端
- 文件上传使用浏览器 File API，支持图片预览和视频播放
